Project Timeline Tool — How to Use

A guide to the Project Timeline Collapse Drop Down tool · Thai Smile Bus / EMMA project

This tool turns a simple list of milestones into a clean, presentation-ready timeline that you can edit live during meetings, then export to a PowerPoint slide or PDF. It is one self-contained file — no installation, works offline, opens in any browser.

1 Opening the tool

  • Double-click project timeline collapse drop down.html — it opens in your web browser (Chrome or Edge recommended).
  • The top half is the editor (a table you type into). The bottom half is the timeline, which redraws instantly every time you change something.
  • Click the big title at the top to rename the project.

2 The milestone table

Each row is one update. The columns are:

ColumnWhat to put
Datedd/mm/yyyy (e.g. 05/06/2026). Thai Buddhist years like 2569 convert automatically. Leave blank for "not scheduled yet".
DescriptionThe milestone name. This is the identity — rows with the same description merge into one box (see §4).
StatusPlan / Done / Delayed (or your own). Sets the color of the dot, box border, and date.
OwnerWho is responsible (KING GEN, INNOXPLAN, MTEC-NECTEC, …). Shown as a tag — with a logo if you upload one (see §5).
PhaseCategory (Meetings, Procurement, Testing, …). Sets the small colored strip on the box and the phase legend.
NotesExtra detail, shown small inside the box.

Row controls on the right:

  • ▲ / ▼ — move the row up or down.
  • ✕ — delete the row.
  • + Add milestone (top toolbar) — adds a new blank row.

3 Adding new dropdown values

The Status, Owner, and Phase dropdowns each have an ➕ Add new… entry at the bottom. Pick it to create a value on the spot:

  • Owner / Phase — type the name in the popup. Done.
  • Status — type the label, then a small color menu appears; click a color and Confirm. The new status colors its dots, boxes, and dates everywhere.
New values become available in every row's dropdown, and are saved inside your project file (see §8).

4 Grouping milestones with the same description

If several rows share the same Description, the timeline shows them as one box that stacks each date → status line vertically.

  • Example: Meeting 1 — 01/01/2026 — Plan and Meeting 1 — 02/01/2026 — Done → a single "Meeting 1" box listing both dates.
  • The box sits at the earliest date; its dot/border use the latest status's color.
  • A unique description simply shows as a normal single-line box.

5 Company logos

Open the Company logos panel (under the editor). It lists every owner you use.

  1. Make sure the company is set in a row's Owner column.
  2. In the logos panel, find that company and click 📷 to upload its logo image.
  3. Every box with that owner now shows the logo next to the company name, automatically.
  • Upload once per company — not per row.
  • Click ✕ in the panel to remove a logo.
Logos are embedded inside the file/project — they travel with Save/Load, Export PNG, Print, and the online link. No separate image files to manage.

6 Fitting everything on one page (Rows)

A long timeline in a single line becomes tiny on a slide. Use the Rows dropdown in the toolbar to wrap it into stacked rows:

  • 2 rows (default) — best for ~15–20 milestones on a landscape A4 / 16:9 slide. Boxes are larger and readable.
  • 3 rows — if you have many milestones and 2 still feels cramped.
  • 1 row — the classic single horizontal line.

Reading order is row 1 left-to-right, then row 2, in date order.

7 Moving things by hand (drag)

The layout arranges itself automatically, but you can fine-tune anything like in PowerPoint:

  • Move a box — drag the box.
  • Move a dot — drag the dot left/right along its line.
  • Move a connector end — drag the small circle where the dotted line meets the box.
  • Pan the whole chart — drag an empty area of the background.
  • Zoom — mouse wheel / trackpad scroll over the chart.
  • Reset view — double-click the empty background.
  • ↺ Reset layout — undo all manual moves and return to auto-arrange.
Your manual positions are saved with the project (§8) and appear in the exported image and print.

8 Saving & reloading your work

The tool keeps one project as one small .json file:

  • 💾 Save JSON — downloads project_timeline.json with everything: milestones, custom statuses/owners/phases, logos, row count, and manual layout.
  • 📂 Load JSON — pick a saved file to restore that project exactly.
Editing in the browser is not auto-saved. Click Save JSON before closing, and keep that file to continue later.

Tip: keep one JSON per project (e.g. EMMA-2026.json). The HTML tool stays the same; only the JSON changes.

9 Export PNG (image for PowerPoint)

  • Click 🖼 Export PNG → it downloads project_timeline_chart.png.
  • The image is cropped to the chart (no empty margins), at 2× resolution, on a white background, with logos included.
  • Drag-and-drop that PNG straight onto a PowerPoint slide.

10 Print / PDF

  • Click 🖨 Print / PDF (or Ctrl/Cmd + P).
  • Choose Landscape, and "Save as PDF" as the destination to make a shareable PDF.
  • The editor and toolbar are hidden automatically — only the title and timeline print.
If the timeline looks small when printing, set the Rows dropdown to 2 or 3 first so it fills the page.

11 Sharing via GitHub (online link)

The tool is hosted so others can open it in a browser, no files needed:

https://elonmark11.github.io/Bua/project%20timeline%20collapse%20drop%20down.html

To update the online version after you change the file:

  1. Go to the Elonmark11/Bua repository on github.com.
  2. Open the file, drag your updated HTML in (or use "Upload files"), keeping the same name.
  3. Click Commit changes. The link refreshes within a minute or two.
The link shows the tool. To share a specific project's data, also send the matching .json file (the recipient opens the link, then Load JSON).

12 Troubleshooting

ProblemFix
Changes disappeared after closingYou didn't Save JSON. Always Save before closing; reload with Load JSON.
Timeline too small on the slideIncrease the Rows dropdown (2 or 3) before exporting/printing.
A logo doesn't showCheck the row's Owner name matches the one you uploaded the logo for (spelling/case). Re-upload in the logos panel if needed.
Boxes overlap after draggingClick ↺ Reset layout to return to auto-arrange.
Two milestones merged unexpectedlyThey have the exact same Description. Make the descriptions different to split them.
Online link looks oldRe-upload the file to GitHub (§11) and wait a minute.
